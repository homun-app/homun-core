# Installer Signing Setup Guide

> **Who this is for**: the Homun maintainer configuring code signing for the native installer pipeline. This doc explains how to go from "I have an Apple Developer account" to "the CI pipeline produces signed + notarized releases automatically".
>
> **Status**: Sprint 8 scaffolded the workflow with graceful fallback to unsigned artifacts. Follow this doc to enable the signed path.

---

## TL;DR checklist

- [ ] **macOS**: export `.p12` from Keychain Access, create app-specific password, add 6 GitHub Secrets, push a tag
- [ ] **Windows**: _deferred post-v1.0_ — requires purchasing an Authenticode EV certificate + HSM (see bottom of this doc)

---

## macOS — Apple Developer ID signing + Notarization

### 1. Confirm cert type in your Apple Developer account

You need a **Developer ID Application** certificate, not App Store or Mac App Store. Log into [developer.apple.com](https://developer.apple.com/account/resources/certificates/list) and verify you have a cert of type **"Developer ID Application"** listed under Certificates.

If you don't, create one:
- Certificates → `+` → Production → "Developer ID Application"
- Upload a CSR generated by Keychain Access (Keychain Access → Certificate Assistant → Request a Certificate From a Certificate Authority, save to disk)
- Download the `.cer` file and double-click to install in your login keychain

### 2. Export the certificate as a `.p12`

1. Open **Keychain Access** (Applications → Utilities)
2. Switch to "login" keychain in the sidebar
3. Find your "Developer ID Application: Your Name (TEAMID)" certificate
4. Expand it (click the ▶) to reveal the private key underneath
5. Select both the certificate AND the private key (Cmd-click)
6. Right-click → **Export 2 items...**
7. Save as `homun-signing.p12`, File Format: Personal Information Exchange (.p12)
8. Set a strong password when prompted — you'll need this below as `APPLE_CERTIFICATE_PASSWORD`

### 3. Create an app-specific password

1. Go to [appleid.apple.com](https://appleid.apple.com) and sign in
2. Under "Sign-In and Security", click **App-Specific Passwords**
3. Click `+`, label it "Homun notarytool CI"
4. Copy the generated password — it looks like `abcd-efgh-ijkl-mnop`
5. This is `APPLE_APP_SPECIFIC_PASSWORD` below. You **cannot** view it again, so store it immediately in your password manager.

### 4. Find your Team ID and Signing Identity string

**Team ID** (10-char alphanumeric like `A1B2C3D4E5`):
- developer.apple.com → Membership page, "Team ID" field

**Signing Identity** (used by `codesign --sign`):
```bash
security find-identity -v -p codesigning | grep "Developer ID Application"
```
Example output: `1) ABC123...  "Developer ID Application: Fabio Cantone (A1B2C3D4E5)"`
Use the **quoted string** (everything inside the quotes) as `APPLE_SIGNING_IDENTITY`.

### 5. Configure GitHub Secrets

Go to `github.com/homun-app/homun-core` → Settings → Secrets and variables → Actions → New repository secret. (Secrets live on the **private** repo where CI runs; the `release.yml` workflow will publish signed artifacts to `homun-app/homun` via the cross-repo PAT — see `docs/MIGRATION-SPLIT-REPO.md` Phase 5-6.)

Add these 6 secrets:

| Secret name | Value |
|---|---|
| `APPLE_CERTIFICATE_P12` | Base64-encoded contents of `homun-signing.p12`. Generate with `base64 -i homun-signing.p12 \| pbcopy` (macOS). |
| `APPLE_CERTIFICATE_PASSWORD` | The password you set in step 2 |
| `APPLE_SIGNING_IDENTITY` | The quoted string from step 4 |
| `APPLE_TEAM_ID` | 10-char Team ID from step 4 |
| `APPLE_ID` | Your Apple ID email address |
| `APPLE_APP_SPECIFIC_PASSWORD` | The password from step 3 |

### 6. Test locally first (recommended)

Before running CI, test the signing path on your own machine. In the repo root:

```bash
# Build a release binary
cargo build --release

# Set env vars in your shell (don't paste the password into your shell history — use a password manager)
export APPLE_SIGNING_IDENTITY="Developer ID Application: Your Name (TEAMID)"
export APPLE_TEAM_ID="YOURTEAMID"
export APPLE_ID="you@example.com"
export APPLE_APP_SPECIFIC_PASSWORD="abcd-efgh-ijkl-mnop"

# Run the packaging script
HOMUN_ARCH=arm64 bash packaging/macos/create-dmg.sh
```

Expected output includes:
- `✓ Bundle signed`
- `--- Notarizing ---`
- `✓ Notarized + stapled`
- Output DMG in `packaging/macos/build/Homun-1.0.0-arm64.dmg`

Verify the result:

```bash
spctl --assess --type open --context context:primary-signature -v packaging/macos/build/Homun-*.dmg
# Should print: accepted
# source=Notarized Developer ID
```

If this passes locally, the CI will work.

### 7. Push a tag and watch CI

```bash
git tag -a v0.1.1 -m "First signed release"
git push origin v0.1.1
```

The `release.yml` workflow kicks in. Watch it at `github.com/homun-app/homun-core/actions`. The `package-macos` job should complete in ~8-15 minutes (binary build + sign + notarize + staple).

If the "unsigned warning" is appended to the release notes after the workflow finishes, something in the secrets is misconfigured — check that `APPLE_CERTIFICATE_P12` is set (and not empty) and re-run the workflow from the Actions tab.

## Windows — Authenticode signing (deferred post-v1.0)

### The short version

Windows native `.msi` packaging is not enabled in v1.0. Windows users install via WSL2 — see [`INSTALL-WINDOWS-WSL.md`](./INSTALL-WINDOWS-WSL.md).

### When we will enable it

When one of these conditions becomes true:
- Homun has enough Windows-paying users to justify $400-600/year in cert fees
- A sponsor/donor offers to fund the cert and HSM
- We qualify for and obtain a free community OSS cert (see "Free signing via SignPath" below)

### Cost breakdown

| Item | Cost | Notes |
|---|---|---|
| OV (Organization Validation) cert | $200-400/year | SmartScreen reputation takes ~3000 downloads to mature |
| EV (Extended Validation) cert | $400-600/year | SmartScreen reputation is immediate |
| HSM hardware token | $100-200 one-time | Required by CA/Browser Forum baseline post-2023 |
| Azure Key Vault cloud signing (alternative) | ~$100/month | Lets CI sign without HSM access on runner |

**Total cold start**: ~$600/year (OV path) or ~$900 first year / ~$600/year after (EV path).

### Free signing via SignPath Foundation

[SignPath](https://signpath.org) offers **free code signing for OSS projects** meeting these criteria:
- Project is **OSI-approved** open source (sorry, PolyForm-Noncommercial is not OSI-approved)
- Public repo with verifiable build process
- Manual review + approval by SignPath volunteers

**Blocker for Homun**: `PolyForm-Noncommercial-1.0.0` is not on the [OSI approved list](https://opensource.org/licenses/). SignPath is therefore unavailable unless Homun relicenses.

### Commercial CAs that sell Authenticode certs

| CA | OV price range | EV price range | Notes |
|---|---|---|---|
| DigiCert | $400/year | $700/year | Premium, widely trusted, offers cloud signing |
| Sectigo | $200/year | $400/year | Cheaper, same trust level |
| GlobalSign | $350/year | $500/year | Strong enterprise support |
| SSL.com | $250/year | $350/year | Cheapest EV, smaller player |
| Certum | $100/year | n/a | Cheapest OV, EU-based, lighter validation |

### When you pull the trigger

1. Purchase cert from a CA above (expect 3-10 business days for identity verification on OV, longer for EV)
2. Receive cert on HSM USB token (or cloud-hosted via CA portal if you chose cloud signing)
3. Add Windows packaging scaffold to `packaging/windows/` — see that dir's README for what needs to go in there
4. Add `package-windows` job to `.github/workflows/release.yml` — wire it similarly to `package-macos` with conditional signing
5. Set GitHub Secrets:
   - `WINDOWS_CERT_PFX` (base64, if using PFX)
   - `WINDOWS_CERT_PASSWORD`
   - Or for HSM: cannot use GitHub-hosted runner — set up self-hosted Windows runner with HSM USB connected
6. Update `docs/INSTALL-WINDOWS-WSL.md` to mention the new native installer as an alternative to WSL (both supported paths)

## Rotating / revoking credentials

### macOS cert expires (annual)

Apple Developer ID certs are valid for 5 years, so rotation is infrequent. When the renewal comes:

1. Generate new cert following steps 1-2 above
2. Update `APPLE_CERTIFICATE_P12` and `APPLE_CERTIFICATE_PASSWORD` in GitHub Secrets
3. Update `APPLE_SIGNING_IDENTITY` **only if the Team ID changed** (usually not)
4. Old signatures on existing releases remain valid until the cert actually expires — don't re-sign old releases

### App-specific password compromised

Delete the compromised password at appleid.apple.com → App-Specific Passwords, create a new one, update `APPLE_APP_SPECIFIC_PASSWORD` in GitHub Secrets.

### Cert is compromised

1. **Immediately revoke** at developer.apple.com → Certificates → delete the cert
2. Apple will blacklist all binaries signed by that cert in the next Gatekeeper update
3. Issue a new cert + update secrets
4. Communicate the revocation in release notes so users know to re-download

### GitHub Secret gone wrong

If you accidentally commit a secret or it ends up in a log:

1. Delete the secret from GitHub Secrets
2. Rotate the underlying credential (p12 password, app-specific password)
3. Update GitHub Secret with new value
4. Scan git history: `git log --all -p | grep -i "BEGIN CERTIFICATE\|p12\|apple_cert"` — if anything leaked, force-push a cleaned history (destructive, only do if truly necessary)
