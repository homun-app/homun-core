# Post Task Memory Flush

## Overview

A task is not complete until future agents can understand what changed, why, and what remains.

## Flush Checklist

Update affected artifacts:

- `docs/roadmap.md`: current phase, milestones, blockers, next action.
- `docs/wiki/session-state.md`: current state and last verification.
- `docs/wiki/thread-ledger.md`: chat handoff when work should continue in another conversation.
- `docs/wiki/decisions.md`: durable choices and tradeoffs.
- `docs/wiki/sources.md`: Context7 or official-doc evidence.
- `docs/wiki/architecture.md`: structure or responsibility changes.
- `docs/wiki/code-standards.md`: durable local standards.
- `docs/wiki/open-questions.md`: unresolved ambiguity.
- Folder `README.md` or `FOLDER.md`: changed folder responsibilities.

## Write Policy

- Write concise durable facts.
- Do not dump transcripts.
- Ask before saving personal preferences to global memory.
- Do not store secrets or credentials.

## Completion Gate

Before final response, confirm whether memory/roadmap artifacts were updated or explain why no update was needed.
