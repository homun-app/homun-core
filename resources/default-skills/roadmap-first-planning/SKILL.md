# Roadmap First Planning

## Overview

The roadmap is the agent's operational memory. Non-trivial work must have an active goal, plan, blockers, and next action before implementation.

## Planning Gate

Before implementation:

1. Read or create `docs/roadmap.md`.
2. Read the relevant wiki pages and folder docs.
3. Create or update a plan in `docs/plans/YYYY-MM-DD-topic.md`.
4. Link relevant sources from `docs/wiki/sources.md`.
5. Define verification commands and acceptance criteria.

## Roadmap Template

`docs/roadmap.md` should include:

- Active goal.
- Current phase.
- Milestones.
- Blockers.
- Recent decisions.
- Next action.

## Plan Requirements

Plans must include:

- Scope and non-scope.
- Files or folders likely to change.
- Required docs/sources.
- Test strategy.
- Verification commands.
- Wiki/folder docs that must be updated.

## Stop Conditions

Stop and clarify if the active goal, source evidence, acceptance criteria, or next action is unclear.
